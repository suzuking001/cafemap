🍜 食べ歩きヒートマップ - Playwright テスト環境

🔴 機能 1：結果の解釈支援機能
目的: ヒートマップ生成後に、カテゴリスコアに基づく人間の気持ちいいメッセージを表示する。

テストケース:

✓ カレーが突出している時に「カレー激戦区！」などのメッセージ表示
✓ ラーメンが少ない時に「ラーメンを探しましょう」というメッセージ表示
✓ ストレスのない、ユーモアのあるメッセージ

判定基準:

  • 待機中 generate() の後に解釈メッセージが表示されるか
  • スコアの高いカテゴリのメッセージを表示するか
  • ユーザーに親近感を与えるか
🔴 機能 2：スクリーンショット共有機能
目的: ヒートマップの画像として保存し、SNS で共有できる機能を実装する。

テストケース:

✓ シェアボタンの表示
✓ マップ画面の正確な描画
✓ PNG/画像としてのエクスポート

判定基準:

  • 待機中 「📸 スクリーンショット共有」ボタンの実装確認
  • canvas に正確な地図描画か
  • ファイルダウンロードが正しく動作するか
🔴 機能 3：保存履歴機能
目的: localStorage に以前のヒートマップを保存し、再表示・比較できる。

テストケース:

✓ 「履歴」タブ/セクションの追加
✓ 保存されたデータが正しく localStorage に格納されるか
✓ 古いデータの自動削除（最大 10 件まで）

判定基準:

  • 待機中 localStorage への保存機能の実装確認
  • 履歴表示 UI の実装確認
  • 「復元」ボタンによる再表示か
